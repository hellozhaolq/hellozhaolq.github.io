sudo 指令

-功能： 以 root 的身分执行命令
-语法： sudo 其他指令
-用户： 被 root 加入到『/etc/sudoers』文件中的用户

1．root 的密码除了 root 本人知道外，不需被其他需要用到 root 权限的用户知道。使用 sudo 时，要求输入的密码是『执行 sudo 指令的用户的密码』。

2．把所有可执行 sudo 指令的用户都规范在『/etc/sudoers』这个文件中，root 可以很容易地掌控整个系统。

切换用户

切换用户的命令

su username

从普通用户切换到root用户，还可以使用命令

sudo su

在终端输入 exit 或 logout 或使用快捷方式 ctrl+d ，可以退回到原来用户。

在切换用户时，如果想在切换用户之后使用新用户的工作环境，可以在 su 和 username 之间加 - ，例如 su - root

提示符： $ 表示普通用户；# 表示超级用户。

sudo su - root 切换的 root 用户和 root 登录用户区别

普通用户使用 sudo 来执行只有 root 才能执行权限的命令，跟用 root 用户执行是不一样的，因为这时候它用的还是普通用户的环境变量。

用 sudo su - root 切换的 root 用户启动的是非登录交互式shell， root 登录用户启动的是登录shell，它们加载的环境变量不一样。另外，它们的 uid 也是不一样，只有 euid 是相同的。

apt-cache、apt-get、apt

apt-cache、apt-get 和 apt 都是Linux 下的包管理工具，其中 apt 是 apt-get 和 apt-cache 的集合体，旨在为终端用户提供更统一、用户友好的交互式体验，而 apt-get 和 apt-cache 是更底层的工具，主要用于脚本编写和自动化任务。

apt-get

  • 功能:

    主要用于从仓库安装、升级和卸载软件包。

  • 特点:

    适用于脚本和自动化操作，向后兼容性强，拥有更多底层的选项。

apt-cache

  • **功能:**用于搜索软件包信息，例如查找软件包的详细描述、版本信息和依赖关系。
  • **特点:**专门用于软件包的查询。

apt

  • 功能:

    集成了 apt-get 和 apt-cache 的常用功能，提供了一个集中的界面来管理软件包。

  • 特点:

    • 用户友好:

      提供了更简洁的命令选项和更有效的参数组织。

    • 交互体验:

      在安装或删除程序时，会显示进度条和可升级的软件包数量，适合终端用户交互使用。

    • 统一入口:

      将多个工具的功能整合在一起，使用户无需在不同命令之间来回切换。

总结

  • 新手和日常使用:

    建议使用 apt，它更方便、用户友好，提供了许多实用的交互功能。

  • 脚本和自动化:

    建议使用 apt-get 和 apt-cache，因为它们在脚本中具有更强的兼容性和更多的底层选项，可以确保操作的稳定性和可靠性。
