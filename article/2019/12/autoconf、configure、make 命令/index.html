autoconf 与 make configure 的区别

命令 类型 作用
autoconf 正宗的工具命令 属于 GNU Autotools，用于从 configure.ac 自动生成标准的 configure 脚本。
make configure 初始 Makefile 的 configure 目标 某些项目源代码里原本就附带了一个“手写”或“初始”的 Makefile，其中定义了一个 configure 目标，通常用于调用 autoconf 或相关工具链生成 configure 脚本。
也就是说，make configure 相当于间接调用 autoconf，但它依赖于初始 Makefile 中有没有定义这个目标。

configure

configure 并不是一个通用的命令，而是 autoconf（GNU Autotools）生成的 shell 脚本。它的作用是在不同 Unix/Linux 平台上自动检测系统特性，并生成适合当前环境的构建文件（如 Makefile 和头文件），使源代码可以顺利被编译和链接。

大致工作：

  • 检查系统环境（操作系统和架构、C 编译器、依赖库/头文件、头文件等）
  • 根据提供的选项（如 --prefix、--with-xxx）生成定制化的、符合GNU规范的 Makefile 构建文件
  • 输出配置状态文件（如 config.status、config.h）

通用选项

这些是 autoconf 自动加进去的：

  • --help：显示所有可用选项（最常用）。
  • --prefix=PATH：安装路径前缀（默认是 /usr/local）。
    • 比如 --prefix=/usr，安装后文件会放在 /usr/bin, /usr/lib 等目录。
  • --exec-prefix=PATH：可执行文件安装目录前缀（默认跟 --prefix 相同）。
  • --sysconfdir=PATH：配置文件路径（默认 /usr/local/etc）。
  • --libdir=PATH：库文件路径（默认 /usr/local/lib）。
  • --includedir=PATH：头文件路径（默认 /usr/local/include）。
  • --enable-FEATURE：启用某个可选的软件特性。./configure --enable-debug
  • --disable-FEATURE：禁用某个可选的软件特性。./configure --disable-shared
  • --with-PACKAGE[=DIR]：指定 某个外部依赖包的位置，或者 启用 对该包的支持。./configure --with-ssl=/usr/local/ssl
  • --without-PACKAGE：禁用 对某个外部依赖包的支持。./configure --without-x

程序自定义选项

不同的程序会在 configure.ac 或 configure.in 文件里，定义额外的 --enable-xxx、--with-xxx，所以每个程序支持的选项都不同。

比如：

  • nginx 的 configure 会有 --with-http_ssl_module、--with-stream。
  • Python 的 configure 会有 --enable-optimizations、--with-lto。
  • ffmpeg 的 configure 会有 --enable-gpl、--enable-libx264。

make 或 make all

作用：执行 Makefile 的 all 目标。根据 Makefile 的规则编译源代码（如 .c、.cpp），最终生成的产物是：

  • 可执行文件（如 myapp、main）
  • 静态库文件（.a 文件）
  • 动态库文件（.so 文件）
  • 有时还可能生成中间文件（.o）、头文件、配置文件等

打开 Makefile 文件，前两行内容已说明 Makefile 的默认目标是 all ，所以 make 等价于：make all

make install

作用：执行 Makefile 的 install 目标。将编译好的文件复制到系统目录（如 /usr/local/bin、/usr/local/lib 等）。

make clean

作用：

  • 执行 Makefile 的 clean 目标。删除编译过程中生成的临时文件、目标文件（.o）、中间产物和最终的二进制文件（可执行文件、库等），保留源文件和 Makefile。
  • 有利于重新编译或打包源码

make distclean
