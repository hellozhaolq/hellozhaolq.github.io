RabbitMQ

发表于 2019-10-19 分类于 MQ

官方： https://www.rabbitmq.com/install-windows.html

下载 RabbitMQ Server：

https://github.com/rabbitmq/rabbitmq-server/releases

安装 RabbitMQ Server：

RabbitMQ 需要 Windows 版64位 Erlang 的支持，最新版下载： http://www.erlang.org/downloads ，最新稳定版下载： https://www.erlang-solutions.com/resources/download.html 。

提示： 必须使用管理员账户运行 Erlang 安装程序，否则 RabbitMQ 安装程序所需要的注册表将不存在。然后运行 RabbitMQ 安装程序，它将 RabbitMQ 安装为 Windows 服务，并使用默认配置启动它。

运行 RabbitMQ Server：

自定义环境 RabbitMQ 变量

RabbitMQ 使用其默认设置运行，我们可以 自定义 RabbitMQ 环境 或 编辑配置 。默认设置在某些环境中可能完全够用，例如开发和QA。对于所有其他情况以及 生产部署调优 ，可以在 代理 和 插件中 配置许多内容。

运行 RabbitMQ

RabbitMQ 服务自动启动。可以从 “开始” 菜单 启动**/停止/移除/**重新安装 RabbitMQ 服务。

管理服务

可以在 “开始” 菜单找到 RabbotMQ 的 数据库、日志 和 插件 目录的链接。

“开始” 菜单中还有一个命令提示符窗口的链接，该窗口将在 sbin 目录（D:\Program Files\RabbitMQ Server\rabbitmq_server-3.7.14\sbin）启动。这是运行 命令行工具 最方便的方法。CLI 工具必须 验证本地运行的RabbitMQ节点。这涉及共享秘密文件，必须将其放入用户的正确位置。

升级 Erlang VM

如果您有现有安装并且计划将Erlang VM从32位升级到64位版本，则必须在升级VM之前卸载代理。安装程序无法停止或删除已经安装的不同体系结构的Erlang VM。

端口访问

防火墙和其他安全工具可能会阻止 RabbitMQ 绑定到端口，这将导致 RabbitMQ 无法启动。应确保打开以下端口（绿色端口默认启动）：

4369：epmd，RabbitMQ节点和CLI工具使用的对等发现服务

5672、5671：由基于 TLS 的 AMQP 0-9-1版本使用，1.0版本未使用。AMQP所有版本协议下载。

25672：用于节点间和CLI工具通信（Erlang分发服务器端口），并从动态范围进行分配（默认仅限单个端口，计算方式：AMQP端口+ 20000）。不应公开这些端口，除非确实需要这些端口上的外部连接（例如，群集使用 联合 或CLI工具在子网外的计算机上使用）。有关详情， 请参阅网络指南。

35672~35682：由CLI工具（Erlang分发客户端端口）用于与节点通信，并从动态范围进行分配（计算方式：服务器分发端口+ 10000 ~ 服务器分发端口+ 10010）。有关详情， 请参阅网络指南。

15672：HTTP API客户端，管理UI和 rabbitmqadmin（仅当启用了管理插件时）

61613、61614：没有和使用TLS的STOMP客户端（仅当启用了STOMP插件时）

1883、8883：( 如果启用了MQTT插件，则没有和使用TLS的MQTT客户端

15674：STOMP-over-WebSockets客户端（仅当启用了Web STOMP插件时）

15675：MQTT-over-WebSockets客户端（仅当启用了Web MQTT插件时）

配置 RabbitMQ 使用不同的端口。

默认用户访问权限

代理将创建一个访客用户和密码，未配置的客户端通常会使用这些凭据。默认情况下，只有在以localhost身份连接到代理时才能使用这些凭据，因此你需要在从任何其他机器连接之前采取行动。

有关如何创建更多用户、删除访客用户、允许远程访问访客用户，请参阅有关访问控制的文档。

管理代理

要停止代理或检查其状态，请以管理员方式运行 sbin 目录下 rabbitmqctl.bat 。

停止节点

rabbitmqctl.bat stop

检查节点状态

rabbitmqctl.bat status

有关详细信息，请参见 RabbitMQ CLI工具指南 和 rabbitmqctl 手册页。

服务器日志

服务器日志在故障排除和根本原因分析中至关重要。请参阅 日志记录 和 文件和目录位置 指南，以了解日志文件位置，日志轮换等。

作为服务运行时的故障排除

如果在RabbitMQ作为服务运行时Erlang VM崩溃，崩溃转储没有写入当前目录，那么它将被写入 RabbitMQ 服务器的基目录中的 erl_crash.dump 文件（基目录由 RABBITMQ_BASE 环境变量设置，默认为％APPDATA％\％RABBITMQ_SERVICENAME％ - 否则通常为％APPDATA％\ RabbitMQ）。
